Get a user's profile

This endpoint allows you to retrieve information about a user's profile.

Endpoint

GET https://debuggerduck.api.com/users/{user_id}

Request parameters

Header fields

Header field

Value description

Authorization

A valid access token. Required.

Path parameters

Path parameter

Value description

user_id

The user's ID. Required.

Sample request

curl -X GET "https://debuggerduck.api.com/users/{user_id}" 
-H "Authorization: Bearer {your access token}"

Response parameters

See the Response codes and the sample User object below.

User object

Key

Value type

Value description

name

string

The user's name displayed on their profile.

id

string

The user ID for this user.

href

string

The Web API endpoint link for this user.

img

image object

The user's profile image.

Sample User object


{
"name" : "Anthon Anthony",
"href" : "https://debuggerduck.api.com/users/anthon11",
"id" : "anthon11",
"images" : [ {
   "height" : null,
   "url" : "http://debuggerduck.com/profileimages/anthon11/img.png",
   "width" : null
} ]
}

Send a message

This endpoint allows you to send messages to a DebuggerDuck.

Endpoint

POST https://debuggerduck.api.com/messages

Request parameters

The body of the HTTP request is a JSON. All values are required.

Key

Value type

Value description

sender_id

string

The user ID for this user. Required.

message

string

Contains the user's message. Required.

Sample request


curl -X POST -H "Content-Type: application/json" -d '{
"sender_id": "id",
"message":"hello, duck!"
}' "https://debuggerduck.api.com/messages?access_token={your access token}"

Get a reply from a DebuggerDuck

This endpoint allows you to retrieve replies from a DebuggerDuck by their IDs.

Endpoint

GET https://debuggerduck.api.com/messages/{user_id}/received/{id}

Request parameters

Header fields

Header field

Value description

Authorization

A valid access token. Required.

Path parameters

Path parameter

Value description

user_id

The user's user ID. Required.

id

The ID of the message. Required.

Sample request

curl -X GET "https://debuggerduck.api.com/messages/{user_id}/received{id}" 
-H "Authorization: Bearer {your access token}"

Response parameters

See the Response codes and the sample Reply object below.

Response object

Key

Value type

Value description

date

long

The message creation timestamp, used to order the incoming replies.

blinked

boolean

Information about the DebuggerDuck's reaction to a message.

Sample Response object


{
"date" : "1472799302",
"blinked" : true
}

Response codes and statuses

The HTTP Status Codes used by the DuckAPI

 

HTTP status code

Description

200 OK

The request was successfully completed.

201 Created

A new resource was created.

400 Bad request

The request syntax was invalid

401 Unauthorized

The request did not include an authentication token or the authentication token was expired.

403 Forbidden

The client did not have permission to access the requested resource.

404 Not found

The requested resource was not found.

405 Method not allowed

The request method is known by the server but it was disabled and cannot be used.

409 Conflict

This response is sent when a request conflicts with the current state of the server.

500 Internal server error

The request was not completed due to an internal error on the server side.

503 Service unavailable

The server was unavailable.
